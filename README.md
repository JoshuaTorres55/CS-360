# CS-360
# Event Tracker Mobile Application

## Project Overview

This project is a functional Android mobile application developed as part of a full mobile app development cycle. The goal of the application was to design and implement an event management system that allows users to log in, store event data, and receive SMS notifications. The primary user need addressed by this app is organization and communication. Users need a way to securely manage event-related information while also being notified when necessary. The app was designed to combine usability, security, and functionality within a structured Android environment.

---

## User Needs and UI Design

To support user needs, the application required several essential screens and features. These included a Login screen for authentication, a main event display screen, database integration for persistent data storage, and SMS permission handling for notifications. The user interface was initially designed in Project Two and then implemented in Project Three. 

The UI design focused on clarity, logical navigation flow, and simplicity. Layout elements were grouped intentionally, labels were clearly defined, and screens were structured to reduce confusion. By organizing user actions in a straightforward sequence—login, view events, manage notifications—the design supported intuitive interaction. These design decisions helped create a user-centered experience where functionality aligned with user expectations.

---

## Development Approach

The coding process followed a structured and incremental approach. Core components such as activity setup, manifest configuration, and permission handling were implemented before layering additional features. Java classes were organized to maintain readability, and in-line comments were used to clarify logic where appropriate.

During development, I focused on breaking the application into manageable parts rather than attempting to build all features simultaneously. This included separating login functionality, database setup, and SMS permission logic into distinct steps. This approach can be applied in future projects by ensuring that foundational components are stable before integrating advanced features. Incremental development reduces errors and improves maintainability.

---

## Testing and Validation

Testing was conducted using the Android emulator (AVD). Each feature was tested individually to verify correct functionality, including activity transitions, permission prompts, and basic user input validation. 

Testing is critical because mobile applications rely on multiple interconnected components. Through testing, structural issues were revealed, particularly related to activity configuration and integration. This process reinforced the importance of validating each component before final submission. Continuous testing ensures stability, improves reliability, and strengthens overall application performance.

---

## Innovation and Problem-Solving

One significant challenge involved integrating multiple Android components—activities, permissions, and manifest configurations—into a cohesive structure. Addressing runtime permission requirements for SMS functionality required careful configuration in both the manifest file and Java code. 

Although challenges occurred, working through these issues strengthened my understanding of Android’s architecture and lifecycle management. Overcoming these integration obstacles required reviewing documentation, debugging errors, and methodically adjusting configuration files.

The strongest component of this project was the launch planning and overall project structure. The launch plan demonstrated understanding of platform requirements, permission considerations, and user accessibility. Additionally, the project reflects knowledge of Android development fundamentals, including activity management, UI layout structuring, and permission-based functionality.

This artifact represents growth in mobile application design, structured development, and applying user-centered principles within a technical framework.
